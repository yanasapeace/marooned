OTP bank

Since may 2019 I work as a lead UX designer in OTP bank Russia, which is a part of international OTP Group, located in Budapest. My first product was Daily banking process, which includes all stages of Debit Card usage. I started to decompose this complex, partly offline based process via global Customer Journey Map, describing all stages precisely
Each card in this document is either User action, Expectation from the product, Pain, or Sollution, all of them splitted through the stages of a whole process. Sollution cards are ready to use product backlog tasks
This document describes all stages of using a Debit card, from Awareness to Retention. Some of them needed to be reviewed and fixed, some needed to be replaced, or removed completely

For example, this research showed, that there is an activation process via IVR in the current bank's Debit card usage flow. This stage didn't work properly, generating a lot of refuses. We discovered that the whole activation process was rudemental, and it was easy to cut it off completely, which had a very positive effect
